# MotorBridge 0.3.1 Release Test Note

Date: 2026-05-15
Scope: Python package and Python CLI RobStride control startup sequencing.

## Summary

MotorBridge 0.3.1 is a narrow Python CLI timing release. It does not introduce
new motor modes, new public SDK APIs, or new protocol capabilities.

Changed in 0.3.1:

- Python CLI RobStride `mit`, `pos-vel`, and `vel` now follow the same startup
  sequence as the Rust CLI and WebSocket gateway.
- The CLI disables torque before changing RobStride `run_mode`, verifies
  `run_mode` by reading `0x7005`, re-enables torque, then writes the target.
- This fixes direct Python CLI cases where `pos-vel` could enable the motor but
  not move until Rust CLI or gateway scan/control had prepared the mode.

## Python CLI Checks

Ping:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-06 \
  --motor-id 4 --feedback-id 0xFD \
  --mode ping
```

Read current mechanical position:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-06 \
  --motor-id 4 --feedback-id 0xFD \
  --mode read-param --param-id 0x7019
```

RobStride position mode through Python CLI:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-06 \
  --motor-id 4 --feedback-id 0xFD \
  --mode pos-vel --pos 0.2 --vlim 0.5 --loc-kp 20.0 \
  --ensure-strict 1 \
  --loop 1 --dt-ms 20
```

Expected behavior:

- CLI can switch RobStride into native Position mode without requiring a prior
  Rust CLI or WebSocket gateway scan/control operation.
- `--ensure-strict 1` fails if `run_mode` readback does not match the requested
  mode.

## Release Notes

- Only the Python CLI RobStride control startup timing changed in this release.
- Existing RobStride parameter IDs and mode names remain unchanged.
