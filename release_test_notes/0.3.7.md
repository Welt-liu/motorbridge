# Release Test Notes - 0.3.7

Date: 2026-05-21

## Scope

This release focuses on RobStride scan reliability and operator visibility on
Windows PCAN, while keeping Linux SocketCAN behavior and Damiao scan behavior
stable. It aligns the Rust core, WebSocket gateway, Python CLI/package, C++
metadata, README files, and test notes at version `0.3.7`.

## Key Changes

- Python CLI RobStride scan keeps the Windows-safe one-controller-per-host-ID
  probing model.
- Python CLI RobStride single-host scans now print live results in probe order:
  `hit` or `no reply` appears as each motor ID is checked.
- WebSocket gateway RobStride scan uses request-level `channel` and `model`
  instead of inheriting an unrelated default target.
- WebSocket gateway RobStride scan respects explicitly supplied `feedback_ids`
  exactly and no longer appends unrelated defaults such as Damiao `0x11`.
- WebSocket gateway RobStride scan emits live `scan_progress` events with
  `start`, `probe`, `hit`, `no_reply`, and `done` phases, then still returns the
  final `{"ok":true,"op":"scan"}` response for compatibility.
- WebSocket gateway now implements `batch_scan` for MotorBridge Studio
  compatibility.
- Windows PCAN repeated RobStride scans release any active RobStride session
  before probing and add a short post-close release gap to avoid
  `PCAN_ERROR_INITIALIZE`.
- PCAN errors now include symbolic names for `PCAN_ERROR_INITIALIZE` and
  `PCAN_ERROR_ILLHW`.

## Damiao Regression Guard

Damiao scan remains routed through the existing Damiao scan implementation.
The RobStride-specific Windows PCAN release handling and live progress events
are gated to `vendor=robstride` and should not change Damiao scan behavior.
