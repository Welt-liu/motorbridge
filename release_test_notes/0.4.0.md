# Release Test Notes - 0.4.0

Date: 2026-06-02

## Scope

This release fixes Damiao `dm-serial` whole-arm scanning and telemetry through
the WebSocket gateway, especially on Windows. The gateway now releases active
Damiao sessions before scan probes reuse the serial bridge, requests fresh
Damiao feedback before returning state snapshots, and exposes
`damiao_state_many` for multi-joint browser telemetry.

The release aligns Rust workspace metadata, Python package metadata, C++
metadata, README files, changelog, and test notes at version `0.4.0`.

## Key Changes

- Windows Damiao `dm-serial` scans stop gateway state/parameter streams and
  release active sessions before probing the serial bridge.
- Damiao gateway state snapshots include `motor_id`, `feedback_id`, and
  `model`.
- Damiao gateway state reads request fresh feedback with a bounded timeout
  before returning `pos`, `vel`, and `torq`.
- WebSocket gateway adds `damiao_state_many` for refreshing multiple Damiao
  joints in one logical request.
- `param_stream enabled=false` no longer opens or reconnects hardware sessions.
- RobStride v0.3.8 PP/CSP and v0.3.9 `request_feedback()` semantics remain
  available.

## Damiao `dm-serial` Manual Check

Start the packaged gateway:

```bash
motorbridge-gateway -- --bind 127.0.0.1:9002 --vendor damiao --transport dm-serial --serial-port COM7 --serial-baud 921600 --model 4340P --motor-id 1 --feedback-id 17 --dt-ms 20
```

Whole-arm scan:

```json
{"op":"scan","vendor":"damiao","transport":"dm-serial","start_id":1,"end_id":7,"feedback_base":16,"timeout_ms":120}
```

Multi-joint state:

```json
{
  "op": "damiao_state_many",
  "items": [
    {"motor_id":1,"feedback_id":17,"model":"4340P"},
    {"motor_id":2,"feedback_id":18,"model":"4340P"},
    {"motor_id":3,"feedback_id":19,"model":"4340P"}
  ],
  "timeout_ms": 120
}
```

Expected:

- Whole-arm scan reports every physically present joint online, not only the
  first joint.
- Joints that are not present return no scan hit and `has_value:false` in
  `damiao_state_many`, without failing the whole response.
- `pos`, `vel`, and `torq` update per joint in MotorBridge Studio after scan.
- Clicking `enable` is not required just to make previously scanned joints
  appear online.
