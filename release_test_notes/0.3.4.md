# Release Test Notes - 0.3.4

Date: 2026-05-20

## Scope

This release focuses on RobStride parameter persistence robustness, Damiao
configuration-mode reliability, WebSocket gateway transport correctness, strict
Clippy cleanliness, and documentation alignment.

## Key Fixes

- RobStride `save_parameters()` no longer falsely fails with
  `control ack timeout: comm_type=22` when firmware responds to
  `SAVE_PARAMETERS` with a valid non-status device reply.
- Damiao `ensure_control_mode()` tolerates an initial register-10 read timeout,
  writes the requested mode, and retries readback verification.
- `ws_gateway` `set_id` now honors per-message/default transport selection,
  including Damiao `dm-serial`, instead of accidentally forcing SocketCAN/PCAN.
- Strict Clippy is clean with `cargo clippy --workspace --all-targets -- -D warnings`.

## Documentation

- Added `integrations/ws_gateway/PROTOCOL.zh-CN.md`, a detailed JSON protocol
  manual for every WebSocket `op`, parameter, default, response, and usage flow.
- Added `release_test_notes/ROBSTRIDE_CLI_MANUAL_TEST_COMMANDS.md` with paired
  Rust CLI and Python CLI manual RobStride test commands.

## RobStride Parameter Persistence Manual Check

Use a safe writable parameter such as `0x7017 limit_spd`:

```bash
cargo run -p motor_cli -- --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --mode read-param --param-id 0x7017
cargo run -p motor_cli -- --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --mode write-param --param-id 0x7017 --param-value 0.9 --store 1
cargo run -p motor_cli -- --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --mode read-param --param-id 0x7017
```

Expected:

- Write prints readback near `0.900000`.
- `--store 1` should not fail with `comm_type=22`.
- A later read returns the written value.
- Power-cycle and read again to prove flash persistence.

## WebSocket Gateway Manual Check

Start RobStride gateway:

```bash
cargo run -p ws_gateway --release -- --bind 127.0.0.1:9002 --vendor robstride --channel can0 --model rs-00 --motor-id 1 --feedback-id 0xFD --dt-ms 20
```

Send:

```json
{"op":"robstride_read_param","param_id":"0x7017","type":"f32","timeout_ms":500}
{"op":"robstride_write_param","param_id":"0x7017","type":"f32","value":0.9,"verify":true,"timeout_ms":500}
{"op":"store_parameters"}
{"op":"robstride_read_param","param_id":"0x7017","type":"f32","timeout_ms":500}
```

Damiao `dm-serial` ID update path should be tested with:

```json
{"op":"set_id","vendor":"damiao","transport":"dm-serial","old_motor_id":1,"old_feedback_id":17,"new_motor_id":1,"new_feedback_id":17,"store":false,"verify":true}
```

Use real changed IDs only when prepared to recover via scan.
