# Release Test Notes - 0.3.6

Date: 2026-05-21

## Scope

This release focuses on RobStride scan reliability on Windows PCAN while
keeping Linux SocketCAN behavior stable. It aligns the Python CLI and WebSocket
gateway scan implementations around sequential, exact host/feedback ID probing.

## Key Changes

- Python CLI RobStride scan opens only one controller at a time, preventing
  multiple PCAN receive workers from competing on the same Windows CAN channel.
- Python CLI scan cleanup no longer calls `close_bus()` on unbound probe
  controllers, removing the misleading `controller has no motor` scan footer.
- WebSocket gateway RobStride scan now probes host/feedback IDs sequentially
  and exactly with `ping_with_host_id()` / `get_parameter_with_host_id()`.
- WebSocket gateway TCP accept now uses the standard listener before handing
  sockets to Tokio/Tungstenite, fixing Windows loopback clients that connected
  at TCP level but timed out during WebSocket handshake.
- Already discovered motor IDs are skipped in later host-id passes to avoid
  duplicate results and unnecessary traffic.
- WebSocket gateway now responds to `{"op":"capabilities"}` with
  `gateway_version`, vendor support, and `batch_scan` / `robstride_exact_host_scan`
  feature flags.
- Python package, Rust workspace packages, and C++ metadata are aligned at
  `0.3.6`.

## RobStride Scan Manual Checks

Python CLI, Windows PCAN:

```powershell
C:\Users\tianr\Downloads\debug_MOTORBRIDGE\.venv\Scripts\motorbridge-cli.exe scan --vendor robstride
```

Expected:

- More than one hit can be reported when multiple motors are present.
- The command finishes with `scan done: ...`.
- No `controller_close_bus failed: controller has no motor` message appears.

Linux SocketCAN:

```bash
PYTHONPATH=bindings/python/src python -m motorbridge.cli scan \
  --vendor robstride --channel can0 --start-id 1 --end-id 16 \
  --feedback-ids 0xFD,0xFF,0xFE,0x00,0xAA
```

WebSocket gateway scan:

```bash
cargo run -p ws_gateway --release -- --bind 127.0.0.1:9002
```

Send:

```json
{"op":"scan","vendor":"robstride","channel":"can0","model":"rs-00","start_id":1,"end_id":16,"feedback_ids":"0xFD,0xFF,0xFE,0x00,0xAA","timeout_ms":120}
```

Expected response shape:

```json
{"vendor":"robstride","transport":"socketcan","count":1,"start_id":1,"end_id":16,"hits":[{"probe":1,"via":"ping","feedback_id":253,"device_id":1,"responder_id":254}]}
```

Use `MOTORBRIDGE_WS_TOKEN` when binding to non-loopback addresses.
