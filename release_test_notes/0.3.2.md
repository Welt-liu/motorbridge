# MotorBridge 0.3.2 Release Test Note

Date: 2026-05-18
Scope: RobStride fault-report handling, C ABI diagnostics, and Python binding/CLI diagnostics.

## Summary

MotorBridge 0.3.2 is a RobStride fault-diagnostics patch release.

Changed in 0.3.2:

- RobStride `FAULT_REPORT` frames update only the cached fault report.
- RobStride `FAULT_REPORT` frames no longer overwrite latest motion feedback.
- RobStride `FAULT_REPORT` frames no longer advance the status ACK sequence used
  by control commands.
- C ABI and Python expose cached `fault_raw` / `warning_raw` diagnostics.
- RobStride `clear_error()` clears the local cached fault report after the device
  acknowledges the clear request.

## Verification

Commands run locally:

```bash
cargo test -p motor_vendor_robstride
cargo test -p motor_abi
python -m py_compile bindings/python/src/motorbridge/abi.py bindings/python/src/motorbridge/core.py bindings/python/src/motorbridge/cli.py
```

Expected behavior:

- A RobStride operation-status frame remains the source of position, velocity,
  torque, and temperature feedback.
- A RobStride fault-report frame is queryable as fault diagnostics but does not
  appear as bogus `-720 deg`, `-50 rad/s`, or `0 C` feedback.
- Python code can call `motor.robstride_get_fault_report()` and receive
  `(fault_raw, warning_raw)`.

## Release Notes

- This release changes RobStride fault handling and exposes diagnostics.
- Existing command names and motion-control APIs remain compatible.
