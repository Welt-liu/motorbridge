# MotorBridge 0.3.3 Release Test Note

Date: 2026-05-19
Scope: version reporting, Python binding version API, unified parameter store flow, and RobStride save ACK handling.

## Summary

MotorBridge 0.3.3 aligns CLI and binding version reporting and makes parameter
persistence explicit and consistent across RobStride and Damiao.

Changed in 0.3.3:

- Rust CLI supports `motor_cli -v` / `motor_cli --version`.
- Python CLI supports `motorbridge-cli -v` / `motorbridge-cli --version`.
- Python binding exposes `motorbridge.__version__` and `motorbridge.get_version()`.
- Python `robstride-write-param` and `damiao-write-param` support `--store 1`.
- Rust CLI RobStride `write-param` supports `--store 1`.
- Rust CLI Damiao supports `read-param` / `write-param` with `--type`,
  `--verify`, and `--store`.
- Python CLI disables argparse long-option abbreviation for root, subcommand,
  and legacy parsers.
- RobStride `save_parameters()` now waits for the status ACK after sending
  communication type `22`.

## Verification

Commands run locally:

```bash
cargo fmt
cargo test -p motor_vendor_robstride
cargo test -p motor_cli
cargo test -p motor_abi
python -m py_compile bindings/python/src/motorbridge/__init__.py bindings/python/src/motorbridge/cli.py
```

Expected behavior:

- `motor_cli -v` prints `motor_cli 0.3.3`.
- `motorbridge-cli -v` prints `motorbridge 0.3.3` when installed from the
  0.3.3 package.
- `import motorbridge; motorbridge.get_version()` returns the installed package
  version.
- `robstride-write-param ... --store 1` performs write, optional verify, and
  RobStride save-parameters.
- Invalid subcommand options such as `robstride-write-param --mode save` are
  rejected instead of being interpreted as `--model save`.

## Release Notes

- This release is CLI/API alignment work; motion-control command semantics remain
  compatible.
- Parameter persistence remains explicit: use `--store 1` when non-volatile
  saving is intended.
