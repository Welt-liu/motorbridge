# MotorBridge 0.3.0 Release Test Note

Date: 2026-05-15
Scope: RobStride clear-error, active-report, fault diagnostics, version alignment, Python binding/CLI, Rust CLI, C ABI, WebSocket gateway, and documentation alignment.

## Summary

MotorBridge 0.3.0 extends the 0.2.9 RobStride/Damiao alignment with RobStride commissioning and arm bring-up features.

New in 0.3.0:

- RobStride `clear_error` sends communication type `4` with `data[0]=1`.
- RobStride active report is exposed through Rust CLI, Python CLI/SDK, C ABI, and WS gateway.
- RobStride communication type `21` fault reports are decoded into raw fault/warning words and documented bit fields.
- `EPScan_time(0x7026)` guidance is documented for RobStride active-report bring-up; `3` means about 20 ms.
- Public docs now describe the three hardware paths first: PCAN, CANable candleLight/gs_usb, and Damiao `dm-serial`.

## RobStride Checks

Clear error:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-00 \
  --motor-id 2 --feedback-id 0xFD \
  --mode clear-error --loop 1 --dt-ms 20
```

Set active-report interval to about 20 ms:

```bash
motorbridge-cli robstride-write-param \
  --channel can0 --model rs-00 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7026 --type u16 --value 3 --verify 1
```

Enable active report:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-00 \
  --motor-id 2 --feedback-id 0xFD \
  --mode active-report --active-report 1 \
  --loop 1 --dt-ms 20
```

Control smoke checks:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-00 \
  --motor-id 2 --feedback-id 0xFD \
  --mode pos-vel \
  --pos 1.5 --vlim 1.0 --loc-kp 5.0 \
  --loop 1 --dt-ms 20

motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-00 \
  --motor-id 2 --feedback-id 0xFD \
  --mode mit --ensure-strict 1 \
  --pos 0.5 --vel 0 --kp 20.0 --kd 0.5 --tau 0 \
  --loop 100 --dt-ms 20
```

## Damiao Checks

Standard CAN scan:

```bash
motorbridge-cli scan \
  --vendor damiao --transport socketcan --channel can0 \
  --start-id 1 --end-id 32
```

Damiao serial bridge scan:

```bash
motorbridge-cli scan \
  --vendor damiao --transport dm-serial \
  --serial-port /dev/ttyACM0 --serial-baud 921600 \
  --start-id 1 --end-id 32
```

Damiao parameter read/write entrypoints:

```bash
motorbridge-cli damiao-read-param \
  --channel can0 --model 4340P \
  --motor-id 0x01 --feedback-id 0x11 \
  --param-id 0x0001 --type f32

motorbridge-cli damiao-write-param \
  --channel can0 --model 4340P \
  --motor-id 0x01 --feedback-id 0x11 \
  --param-id 0x0001 --type f32 --value 1.0 --verify 1
```
