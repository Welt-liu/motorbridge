# Release Test Notes - 0.3.8

Date: 2026-05-26

## Scope

This release focuses on RobStride position-control correctness and cycle-time
behavior. It adds dedicated PP and CSP position-control paths, keeps the legacy
unified `pos-vel` mapping compatible, and removes the default blocking wait from
RobStride parameter writes.

The release aligns Rust workspace metadata, Python package metadata, C++
metadata, README files, changelog, and test notes at version `0.3.8`.

## Key Changes

- Added RobStride `pos-vel-pp`:
  `run_mode=1 -> enable -> vel_max(0x7024) -> acc_set(0x7025) -> loc_ref(0x7016)`.
- Added RobStride `pos-vel-csp`:
  `run_mode=5 -> enable -> limit_spd(0x7017) -> loc_ref(0x7016)`.
- Added C ABI exports and Python binding methods:
  `robstride_send_pos_vel_pp()` and `robstride_send_pos_vel_csp()`.
- Added Rust CLI and Python CLI support for `pos-vel-pp` and `pos-vel-csp`.
- Kept legacy RobStride `pos-vel` behavior compatible.
- RobStride parameter writes now default to no status-ack wait. To restore the
  old conservative wait, set:
  `MOTORBRIDGE_ROBSTRIDE_WRITE_ACK_TIMEOUT_MS=260`.
- Added `bindings/python/examples/robstride_posvel_timing_demo.py` for timing
  comparison across legacy `pos-vel`, PP, and CSP.
- Documented direct Python calls and the recommended prepared high-rate loop:
  initialize PP/CSP once, then cyclically write only `loc_ref(0x7016)`.

## RobStride Hardware Timing Check

Default no-ack parameter writes:

```bash
PYTHONPATH=bindings/python/src \
python3 bindings/python/examples/robstride_posvel_timing_demo.py \
  --channel can0 \
  --model rs-00 \
  --motor-id 1 \
  --feedback-id 0xFD \
  --mode all \
  --profile both \
  --pos 0.0 \
  --vlim 1.0 \
  --acc 10.0 \
  --loop 100 \
  --dt-ms 10
```

Conservative ack comparison:

```bash
PYTHONPATH=bindings/python/src \
python3 bindings/python/examples/robstride_posvel_timing_demo.py \
  --channel can0 \
  --model rs-00 \
  --motor-id 1 \
  --feedback-id 0xFD \
  --mode legacy \
  --pos 0.0 \
  --vlim 1.0 \
  --loop 20 \
  --dt-ms 10 \
  --ack-timeout-ms 260
```

Expected:

- `legacy` runs close to the requested outer-loop period when ack wait is `0`.
- `pp-full` and `csp-full` are slower per call because each call includes
  `enable` and waits for communication type 3 status ack.
- `pp-prepared` and `csp-prepared` measure the intended high-rate pattern:
  setup once, then cyclic `loc_ref(0x7016)` writes.
